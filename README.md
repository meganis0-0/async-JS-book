## Требования

### Node.js (для запуска `.js` файлов через терминал)

1. Скачай Node.js с официального сайта: **[https://nodejs.org](https://nodejs.org)**
2. Выбери версию **LTS** (рекомендуемая для большинства пользователей)
3. Установи и следуй инструкциям установщика
4. Проверь установку в терминале:

```bash
node --version
```

Если видишь версию (например `v20.11.0`) — всё готово.

### VS Code + Live Server (для запуска `.html` файлов)

1. Установи **[VS Code](https://code.visualstudio.com/)**
2. Открой VS Code → раздел **Extensions** (иконка кубиков или `Ctrl+Shift+X`)
3. Найди и установи расширение **Live Server** (автор: Ritwick Dey)
4. После установки в правом нижнем углу VS Code появится кнопка **Go Live**

## Как запускать файлы

### Через Node.js (терминал)

Используется для **`.js` файлов**. Открой терминал (PowerShell, cmd, bash) и выполни:

```bash
# Перейди в папку репозитория
cd путь/до/async-JS-book

# Запусти нужный файл
node 1.1/sync-code.js
```

Результат выполнения появится в терминале.

> **Важно:** HTML-файлы нельзя запустить через Node.js — для них используй Live Server.  
> Некоторые `.js` файлы используют браузерные API (`fetch`, `performance`, `document`) — их тоже нужно открывать в браузере.

### Через Live Server в VS Code

Используется для **`.html` файлов** (и `.js` файлов, которые работают в браузере).

1. Открой нужный `.html` файл в VS Code
2. Нажми правой кнопкой мыши на файл → **Open with Live Server**  
   _или_ нажми кнопку **Go Live** в правом нижнем углу
3. Браузер откроется автоматически и покажет страницу
4. Вывод `console.log()` смотри в **DevTools** → вкладка **Console** (`F12` в браузере)

### 📁 1.1 — Синхронный код

> Все файлы этой папки запускаются через **Node.js**.

```bash
node 1.1/sync-code.js
```

Три функции, выполняющиеся последовательно. Показывает пошаговое синхронное выполнение.

#### `1.1/blocked.js`

```bash
node 1.1/blocked.js
```

Функция `heavyOperation` — искусственная задержка через цикл (`Date.now()`). Демонстрирует блокировку основного потока.

#### `1.1/sync-problem.js`

```bash
node 1.1/sync-problem.js
```

Симуляция синхронного сетевого запроса. Показывает, почему синхронная сеть заблокировала бы браузер.

> **Примечание:** этот файл зависит от `heavyOperation` из `blocked.js`. Если увидишь ошибку `ReferenceError`, запускай вместе: `node -e "$(cat 1.1/blocked.js 1.1/sync-problem.js)"` — или изучай файлы вместе как концепцию.

### 📁 2.1 — Асинхронность

> `.js` файлы запускаются через **Node.js**.

```bash

```

Синхронная блокировка vs `setTimeout(..., 2000)`. Код продолжает выполняться сразу после планирования таймера.

```bash

```

`setTimeout` на 1 секунду. Вывод: сначала "конец синхронного кода", потом — колбэк таймера.

```bash

```

Смесь синхронных логов, `setTimeout(1000)`, `setTimeout(0)` и `Promise.resolve().then`. Порядок: синхронно → микрозадачи → макрозадачи.

#### `2.1/visualization,js`

> **Внимание:** файл назван с запятой вместо точки — это опечатка в имени файла. Запуск через Node.js:

```bash
node "2.1/visualization,js"
```

#### `2.1/fetch-ex1.js`

```bash
node 2.1/fetch-ex1.js
```

`fetch().then().then()` — неблокирующий сетевой запрос. Нумерованные логи показывают, что синхронный код выполняется до обработки ответа.

> **Примечание:** `fetch` доступен в Node.js начиная с версии **18+**. Если версия ниже, запускай этот файл в браузере через консоль DevTools: вставь код в консоль (`F12` → Console).
